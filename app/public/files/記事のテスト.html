調査事項

  • tRPCとは
    • tRPCはTypescript専用のGraphQLの認識
    • スキーマや型生成を別途用意しなくても簡単に型安全なAPIを構築できるフレームワークです。
    • tRPCはTypescriptでしか使用できず、汎用的なフレームワークではありません
    • 現在Typescriptを使用した型安全のAPIで主流なのはGraphQLだが、言語に関係なく使用できるGraphQLはTypescriptの能力を引き出しきれないという思想からtRPCが生まれました。
  • tRPCのinputとqueryとmutationの使用方法
    • inputはクライアントからの入力情報のバリデーションチェックを行うときに使用します。
    • また、このinputを使用したバリデーションを挟むことで後続の処理でクライアントからの取得情報に対して自動的に型が付与されます。
    • 型のチェックは自前の処理でもライブラリでもいいのですが、tRPCのドキュメントではZodを推奨しているので今回はZodを使用しています。
    • なお、outputというメソッドもあり、これはtRPCが返す値の型を指定することができます。
  • Zodの概要
  • Auth0のAPIでオーディエンスを含める意味
    • これはAuth0というよりOAuthの調査範囲かも
    • オーディエンスはアクセストークンを使用する受け手のURLを記載
    • これがないとAuth0はアクセストークンのpayloadに情報を持たないが、なぜこれがないだけで情報を返さないのかは謎
  • Zodの使用方法
  • Auth0のTerraform解説
    • どこまで解説する？
    • ハードコーディング許容するか
      • いやtfvarsは使用しよう
    • tfstateは解説したい。
    • Auth0とは→Terraformとは→ドキュメント(https://registry.terraform.io/providers/auth0/auth0/latest/docs/guides/quickstart)通りにやる→tfvars使用する→tfstateをドキュメント(https://docs.gitlab.com/ee/user/infrastructure/iac/terraform_state.html)通りにやる
    • 環境自体はドキュメント(https://auth0.com/docs/quickstart/webapp/nextjs/01-login)に従って行うが、APIsにAudienceは追加できるようにする。

tRPCの設定

npm install @trpc/server @trpc/client @trpc/react-query @trpc/next @tanstack/react-query zodを実行して、必要なモジュールをインポートしてください。

tsconfig.jsonのcompilerOptions内にあるstrictもしくはstrictNullChecksをtrueにしてください。

src/serverフォルダ内にcontext.tsを作成し、下記のコードを記載します。

import * as trpc from '@trpc/server';
import { inferAsyncReturnType } from '@trpc/server';
import * as trpcNext from '@trpc/server/adapters/next';

export async function createContext({ req, res }: trpcNext.CreateNextContextOptions) {
    async function configPayload() {
        console.log('context test')
    }
    return configPayload()
}

export type Context = inferAsyncReturnType<typeof createContext>

ここでアクセストークンを取得処理を行いますが、まだAuth0の設定が完了していないので一旦はこのままにします。

同じくserverフォルダにtrpc.tsを作成し、下記のコードを書きます。

import { Context } from "./context";

export const t = initTRPC.context<Context>().create()

src/server/routesフォルダを作成し、その中に下記のコードが記載された_app.tsを作成してください。

import { t } from "../trpc"

const methods = {}

export const appRouter = t.router(methods)

export type AppRouter = typeof appRouter

基本的にmehods内に作成した処理をインポートする形にして呼び出すようにしています。

ためしにserverフォルダ内にservicesフォルダを作成して、hello.tsに下記のコードを作成してください。

import { z } from "zod";
import { t } from "../trpc";

export const hello = t.procedure
　.input((z.object({ name: z.string() })))
    .query(({input}) => {
        return `${input.name} hello`
    })

上記処理は呼び出す時に渡す値に下記のオブジェクトの型しか許容しないようにし、

{
    name:string;
}

渡す値の型が適切な時query内の処理を実行するようになっています。

作成したhelloメソッドを_app.ts内のmethods内に渡してください。

これでtRPCのバック側の準備ができました。

次にsrc配下にutilsフォルダを作成して、tRPCの設定コードを記載したtrpc.tsファイルを作成してください。

import { AppRouter } from '@/server/routers/_app';
import { httpBatchLink } from '@trpc/client';
import { createTRPCNext } from '@trpc/next';
import { NextPageContext } from 'next';
function getBaseUrl() {

    // assume localhost
    return `http://localhost:${process.env.PORT ?? 3000}`;
}

export const trpc = createTRPCNext<AppRouter, NextPageContext>({
    config({ ctx }) {
        return {
            links: [
                httpBatchLink({
                    /**
                     * If you want to use SSR, you need to use the server's full URL
                     * @link https://trpc.io/docs/ssr
                     **/
                    url: `${getBaseUrl()}/api/trpc`,

                    // You can pass any HTTP headers you wish here
                    async headers() {
                        return {
                            // authorization: accessToken.accessToken,
                        };
                    },
                }),
            ],
        };
    },
    /**
     * @link https://trpc.io/docs/ssr
     **/
    ssr: false,
});

src/pagesフォルダ内の_app.tsxにexport default trpc.withTRPC(MyApp);を追加してください。

なお、MyAppの部分がexport default function ...となっていたらアロー関数の変数に変更しておいてください。

以上が完了すればNextJsでtRPCを呼び出せるようになります。

そこでpagesフォルダ内のindex.tsxを次のように変更してください。

const inter = Inter({ subsets: ['latin'] });

export default function Home() {

trpcで呼び出す時は補完が効き、useQueryに必要な引数も補完が効くためとても呼び出しやすくなっています。

この状態でnpm run devを実行して、http://localhost:3000にアクセスすると画像のように文字列が表示されていると思います。

これでNextJsとtRPCの連携は完了しました。

Markdownエディタの設定

styled-componentsのインストールをします。

styled-componentsは以下のコマンドでインストールします。

npm i -D styled-components@5.3.10 @types/styled-components

参考資料(https://github.com/styled-components/styled-components/issues/3998)

package.jsonに以下の記述を追加します。

"resolutions": {
    "styled-components": "^5"
  },

次にMarkdownエディタを使用するために次のコマンドを実行します。

npm i @uiw/react-md-editor next-remove-imports

react-md-editorはmarkdonwエディタで、next-remove-importsはnode_modules以下のCSSなどをインポートしてしまうことによる実行ができなくなる不具合の解消のためにインポートしています。

next.config.jsの拡張子をmjsに変更し、下記のコードを記載してください。

import removeImports from 'next-remove-imports'
const removeImportsFun = removeImports({

});

export default removeImportsFun({
  reactStrictMode: true
})

mjsファイルに変更したのは、react-md-editorの公式ドキュメントでの手順を行うと**require() of ES modules is not supported.**というエラーが発生し、require関数が使用できなかったので、importでも実行できるように拡張子を変更することで対応しました。

ただ、この辺あまり理解していないので、教えていただけると幸いです。

pagesフォルダにpostフォルダを作成したら、post.tsファイルに次の記載をしてください。

import "@uiw/react-md-editor/markdown-editor.css";
import "@uiw/react-markdown-preview/markdown.css";
import dynamic from "next/dynamic";
import { useState } from "react";

const MDEditor = dynamic(() => import("@uiw/react-md-editor"), { ssr: false });
const CreatePost = () => {
  const [value, setValue] = useState<string | undefined>("");

  return (

      <MDEditor value={value} onChange={setValue} height={500} />

  );
};

export default CreatePost;

Prismaの導入

データベースを操作するORM用に下記の手順でprismaを導入します

  1. npm install prisma --save-devを実行し、prismaをインストールします。
  2. npx prisma initを実行し、prismaをセットアップします。
  3. npm install @prisma/clientを実行し、PrismaClientをインストールします。
  4. utilsフォルダにprisma.tsファイルを作成します。
  5. prisma.ts内に下記コードを記載します。
import { PrismaClient } from "@prisma/client";

const prismaServie = new PrismaClient();

export default prismaServie;
  1. prisma.schemaにPostモデルを追加します。
model Post {
  id         Int      @id @default(autoincrement())
  title      String
  content    String
  created_at DateTime @default(now())
  updated_at DateTime @default(now())
}
  1. npx prisma db pushでデータベースを作成したら、npx prisma studioを実行しPostテーブルが存在すればセットアップは完了です。
